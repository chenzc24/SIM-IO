## Step 3a: Netlist Export (si batch)

### Approach

Use Cadence `si` batch netlister — no ADE/Maestro needed. Two SKILL calls + one shell command.

```python
# 1. Generate si.env on remote
client.execute_skill('sh("mkdir -p /tmp/si_run")')
client.execute_skill(
    f'simInitEnvWithArgs("/tmp/si_run" "{lib}" "{tb_cell}" "schematic" "spectre" nil)')

# 2. Run si batch netlister (via SSH shell, NOT via SKILL system())
#    si outputs to /tmp/si_run/netlist
ssh_runner.run_command(
    'cd /tmp/si_run ; '
    'si -batch -cdslib $HOME/cds.lib -command nl')

# 3. Download netlist
ssh_runner.download("/tmp/si_run/netlist", local_path)
```

### Key constraints (from batch-netlist-si.md)

- Use `-command nl` for Spectre (NOT `-command netlist` — causes OSSHNL-510)
- `si` must run via SSH shell, not via SKILL `system()` (would deadlock CIW)
- Cadence env must be sourced first (cshrc with `PATH`, `LM_LICENSE_FILE`)
- Output: single file at `<runDir>/netlist`

### si.env content (auto-generated by `simInitEnvWithArgs`)

```ini
simLibName = "LLM_Layout_Design_Lab"
simCellName = "IO_RING_12x12_tb"
simViewName = "schematic"
simSimulator = "spectre"
simViewList = '("spectre" "cmos_sch" "schematic" "veriloga")
simStopList = '("spectre")
simNetlistHier = t
```

### What `si` produces vs what spectre needs

| Content | si netlist | Ready-to-run deck |
|---------|-----------|-------------------|
| Subcircuit definitions | Yes | Yes |
| Instance connections | Yes | Yes |
| `simulator lang=spectre` | Yes | Yes |
| `global 0` | Yes | Yes |
| Model include | **No** | **Must add** |
| Analysis (tran/dc) | **No** | **Must add** |
| Simulator options | **No** | **Must add** |
| Save signals | **No** | **Must add** |

## Step 3b: Build Simulation Deck

Append simulation commands to the `si`-exported netlist to create a complete spectre deck.

### Template structure

```
// === si-generated circuit netlist (DO NOT EDIT above this line) ===
include "/path/to/PDK/models.scs" section=TT

// Simulator options
simulatorOptions options reltol=1e-4 vabstol=1e-6 iabstol=1e-12 \
    temp=27 tnom=27 gmin=1e-12

// Transient analysis
tran tran stop=10u errpreset=moderate

// Save signals
saveOptions options save=allpub
```

### Configurable parameters

```python
@dataclass
class SimConfig:
    model_include: str           # PDK model file path on remote server
    model_section: str = "TT"    # Process corner
    analysis: str = "tran"       # "tran" | "dc"
    stop: str = "10u"            # tran stop time
    errpreset: str = "moderate"  # "liberal" | "moderate" | "conservative"
    temperature: float = 27.0    # temp parameter
    save_signals: str = "allpub" # "allpub" | "all"
    corner: str = "tt"           # for model section selection
```

### Model include path

Need to know where the PDK model files are on the remote server. Options:
- Hardcode per PDK (user provides)
- Detect via SKILL: `client.execute_skill('simplifyFilename("./models.scs")')`
- Environment variable: `VB_PDK_SPECTRE_INCLUDE` (already used by example scripts)

## Step 3c: Run Spectre

Direct use of existing `SpectreSimulator`:

```python
from virtuoso_bridge.spectre.runner import SpectreSimulator

sim = SpectreSimulator.from_env(
    work_dir=run_dir,
    output_format="psfascii",   # ASCII for easy parsing
)

result = sim.run_simulation(deck_path, {
    "include_files": [model_path],  # uploaded alongside netlist
})
```

`SpectreSimulator` handles:
- Upload netlist + model files to remote server
- Source Cadence cshrc for spectre binary
- Run `spectre -64 netlist.scs +escchars -format psfascii ...`
- Download PSF results + log files
- Parse PSF ASCII into `result.data` dict

## Step 3d: Parse Results

`result.data` from `SpectreSimulator` is already a dict of signal arrays:

```python
result.data = {
    "time": [0.0, 1e-12, 2e-12, ...],
    "DUT.IN0": [0.0, 0.1, 1.8, ...],     # voltages at DUT input
    "DUT.OUT0": [0.0, 0.0, 0.5, ...],     # voltages at DUT output
    "SRC_IN0:PLUS": [0.0, 0.1, 1.8, ...], # source output
}
```

Extract measurements:

```python
def measure_tran(data: dict, signal: str) -> dict:
    """Extract key metrics from a transient signal."""
    time = np.array(data["time"])
    v = np.array(data[signal])

    return {
        "vmax": float(np.max(v)),
        "vmin": float(np.min(v)),
        "vavg": float(np.mean(v)),
        "vpp": float(np.max(v) - np.min(v)),
    }
```

More advanced measurements (delay, slew rate) need two-signal cross-referencing.

## Open Questions

1. **PDK model path on remote server** — need to know where `toplevel.scs` or equivalent is located for the target technology. Could be in `.env` as `VB_PDK_SPECTRE_INCLUDE`.
2. **`cds.lib` path** — si needs this. Detect via SKILL or hardcode per user.
3. **PDK device models** — the DUT (IO ring) contains IO pad cells. Do these have spectre models? Or behavioral models? If only CDL/SPICE models exist, need `spiceIn`-compatible format.
4. **Signal naming in results** — PSF signal names depend on netlisting hierarchy. Need to map `result.data` keys back to pin names (e.g., `"DUT.IN0"` → pin `"D0"`).
5. **si vs direct SKILL read** — if `si` is unavailable or misconfigured, fallback to reading schematic connectivity directly via SKILL and formatting into Spectre netlist manually (see `netlist.md` → "Direct Schematic Read → Netlist").
